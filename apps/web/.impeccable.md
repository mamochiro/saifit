# Saifit — Design Context

## Design Context

### Users
Bangkok gym-goers, 20–40. Beginners to intermediates. Predominantly Thai-speaking. They train at commercial gyms, home setups, or with minimal equipment. They use this app during training — hands chalked, between sets, in low-light gym environments. The job to be done is simple and time-pressured: log the set I just did before I forget what I lifted, then start the rest timer. Every second of friction is a second stolen from the next set.

Primary auth is LINE Login. Most users will never create an email account for a gym app.

### Brand Personality
**stark · precise · earned**

This is a personal trainer's tracking notebook, not a wellness app. It does not congratulate you for showing up — it records what you did. The interface respects effort by getting out of the way.

Voice: direct, compressed, honest. "45 kg × 12" not "Great set! You lifted 45 kg for 12 repetitions."
Anti-references:
- Wellness sage/cream (calm, meditative — wrong energy entirely)
- Neobank teal / startup blue (generic tech)
- Crypto purple glow (decorative darkness, not purposeful darkness)
- Any fitness app that uses green as the primary success color (too expected)

### Aesthetic Direction
**Monochrome dark with pure white as the sole signal color.**

The base is `#09090b` — near-black, barely warm. White (`oklch(97% 0.003 90)`) is the ONLY accent hue. There is no brand color. Visual hierarchy is pure value contrast — dark surface, bright numbers, everything else recedes into the dark.

This is the aesthetic choice of chalk on iron plates. No color needed.

**Theme: Dark** — users train in gyms (low-light) and check the app between sets in dim environments. Dark is not a style choice here; it's the correct environmental answer.

Typography pairing (both by Cadson Demak — Bangkok type foundry, authentic origin):
- **Chakra Petch** — display font for large numbers (weights, reps, timer, PR values, stats). Geometric, slightly technical, precise. Not Thai-capable — used ONLY for Latin/numeric display contexts.
- **K2D** — body font for ALL Thai text and UI labels. Condensed grotesque with excellent Thai script support. Athletic without being aggressive.

Numerals: always `font-variant-numeric: tabular-nums lining-nums`. Numbers must lock to a grid — this is a coaching log, not a social feed.

### Color System (OKLCH)

```css
/* Base surfaces — barely warm (hue 90 tilt toward saffron, not cold blue-gray) */
--color-base:        oklch(10.5% 0.005 90);   /* #09090b equivalent */
--color-surface:     oklch(14% 0.005 90);     /* card / sheet background */
--color-surface-alt: oklch(17% 0.004 90);     /* elevated surface, inputs */
--color-border:      oklch(25% 0.003 90);     /* dividers, borders */
--color-muted:       oklch(40% 0.003 90);     /* inactive, secondary text */
--color-secondary:   oklch(60% 0.002 90);     /* labels, captions */
--color-foreground:  oklch(92% 0.004 90);     /* primary text */

/* Signal — the only "color" in the system */
--color-accent:      oklch(97% 0.003 90);     /* pure white-chalk: CTA buttons, active states */

/* Functional only (never decorative) */
--color-destructive: oklch(62% 0.20 25);      /* errors, delete actions */
```

**The 60-30-10 rule applied:**
- 60% base surfaces (backgrounds, page)
- 30% mid-tones (borders, muted text, inactive elements)
- 10% accent white (complete-set button, active set highlight, PR value)

**Absolute rule:** no other hue enters the system. Not even a tinted ring color, hover tint, or "subtle" background highlight that isn't from this palette.

### Typography Scale

```css
/* Display — Chakra Petch, numbers/stats only */
--text-display-xl:  3rem;    /* rest timer (90s), PR weight (120kg) */
--text-display-lg:  2.25rem; /* workout set weight (45.0 kg) */
--text-display-md:  1.5rem;  /* stat cards (weekly volume) */

/* UI — K2D, all Thai text and labels */
--text-ui-lg:  1.125rem;  /* section headings, exercise names */
--text-ui-md:  1rem;      /* body, set rows */
--text-ui-sm:  0.875rem;  /* labels, captions, secondary info */
--text-ui-xs:  0.75rem;   /* meta, timestamps */

/* Line heights */
--leading-display:  1.0;    /* numbers — no leading needed */
--leading-thai:     1.7;    /* Thai body — ascenders need space */
--leading-latin:    1.5;    /* Latin body */
```

Load via Next.js `next/font/google`:
```ts
import { Chakra_Petch, K2D } from "next/font/google";
const chakraPetch = Chakra_Petch({ subsets: ["latin"], weight: ["400","600","700"], variable: "--font-display" });
const k2d = K2D({ subsets: ["latin", "thai"], weight: ["400","500","600","700"], variable: "--font-body" });
```

Apply in layout: `font-display` on `.font-display` elements, `font-body` as body default.

### Layout Principles

1. **Mobile-first, thumb-zone aware.** Primary action (Complete Set) lives in the bottom-right thumb zone at all times during a workout. No primary action should require reaching above the halfway point of the screen.

2. **Left-aligned numerals, not centered.** Stats and logger rows are left-aligned with right-aligned numbers. Never center a metric with a label floating above it — that's the "hero metric template" and it destroys information density.

3. **Generous vertical rhythm, tight horizontal.** The screen is narrow. Use vertical space generously (24–32px between sections) but never waste horizontal space on decorative margins.

4. **No cards within cards.** Exercise rows within a workout are rows, not cards. A card is a full-bleed unit. Do not nest.

5. **Spacing scale (4pt base):**
   ```
   --space-1: 4px    --space-2: 8px    --space-3: 12px
   --space-4: 16px   --space-6: 24px   --space-8: 32px
   --space-12: 48px  --space-16: 64px
   ```

### Design Principles (apply to every component)

**1. The interface disappears during a workout.**
During an active logging session, no decorative chrome. Screen space belongs to: current exercise name, weight input, reps input, complete button, rest timer. Remove anything that doesn't serve the next set.

**2. Contrast does all the work color cannot.**
No hue does the signaling — only contrast ratio. An active set row is white text on the slightly elevated surface. An inactive set is muted text on the base. That's the whole system.

**3. Numbers are the hero.**
The rest timer at 0:45, the weight at 45.0, the PR at +2.5 kg — these are the emotional moments. Use Chakra Petch, large, tabular. Everything else recedes.

**4. Every tap must feel deliberate.**
Minimum 56px tap targets. 300ms debounce on auto-save. Complete Set at bottom-right. Rest timer docks above keyboard. No accidental triggers during a workout.

**5. Thai first, always.**
All UI text defaults to Thai (K2D). English labels appear only when the user switches locale. Thai line-height is non-negotiable at 1.7 minimum — never clip Thai ascenders.

**6. Earn your feedback.**
Motion means something happened. No ambient animations, no hover glow, no entry shimmer. The rest timer arc depletes smoothly. A PR row brightens. Set completion triggers a momentary visual lock (not a celebration modal). That is the complete motion vocabulary.

### Signature Moments

**Rest Timer (the product's signature moment):**
- Full-width circular arc, drawn in white on the dark surface
- Arc depletes clockwise over the rest duration
- Large tabular numeral (Chakra Petch, 3rem) in the center
- Docked at bottom, above keyboard — NOT centered on screen
- At zero: subtle flash (opacity pulse, 150ms), haptic
- Arc stroke: `oklch(97% 0.003 90)` — full white. No color.
- Arc track: `oklch(25% 0.003 90)` — the border tint

**PR Celebration:**
- No modal, no confetti animation
- The PR value brightens to full white and scales up 1.05× for 300ms
- A small "+PR" badge appears beside the exercise name (no icon, no emoji)
- Then everything returns to rest state
- That's it. Earned, not gifted.

**Complete Set:**
- Button fills to full white on tap
- Set row locks: weight and reps become read-only, foreground goes to muted
- Next empty row activates (slides up 12px, then settles — transform only, no height)
- Whole animation: 200ms ease-out-quart

### Component Patterns

**Set Row:**
```
[Set #] [Weight input] [kg] [×] [Reps input] [✓]
```
- Weight + reps inputs: `inputmode="decimal"`, 56px min-height, full-width on focus
- ✓ button: white background, black checkmark, 56px×56px, rounded-full
- Completed row: muted text, no inputs (replaced with static values), checkmark becomes solid white dot

**Exercise Name:**
- K2D Medium (500), `--text-ui-lg`
- Secondary line: muscle group tags, `--text-ui-xs`, `--color-muted`
- No icon, no colored dot, no category badge

**Bottom Navigation:**
- 4 items: หน้าหลัก / บันทึก / ความก้าวหน้า / ท่า
- White fill for active, muted for inactive
- No label background, no pill highlight — just the icon weight shift

**Stat Card (Progress tab):**
- NOT the hero metric template (big number, label below)
- Instead: number RIGHT-ALIGNED, label LEFT-ALIGNED, single row
- Divider between rows is the border tint, not a line card

### Anti-Patterns (never implement)

- Gradient text (`background-clip: text`) — absolute ban
- Side-stripe borders on cards/alerts (`border-left: Npx solid ...`) — absolute ban
- Rounded cards with generic drop shadows
- Centered metric with floating label above (hero metric template)
- Green checkmarks or green "success" states (too expected for fitness)
- Glowing accents or colored rings on inputs
- Celebration modals for completing a set
- Any hue that isn't `oklch(...% 0.003 90)` — warm-neutral range only
